# Pedido de Ajuda

Quando você estiver travado em um problema e precisar de outro par de olhos, use o sistema de **Pedido de Ajuda**. Ele notifica todos os desenvolvedores disponíveis simultaneamente.

## Solicitando ajuda

1. No Painel, clique no botão **Pedir Ajuda** (canto superior direito).
2. No modal que abre, descreva o contexto do problema:
   - O que você está tentando fazer
   - Onde está travado
   - O que já tentou
3. Clique em **Enviar**.

Um **banner azul** aparece no Painel de todos os outros desenvolvedores com seu nome e a mensagem de contexto.

<Callout type="info">
  O botão **Pedir Ajuda** está disponível apenas para Desenvolvedores e Tech Leads. O Painel continua funcionando normalmente enquanto você aguarda uma resposta.
</Callout>

## Respondendo a um pedido

Quando um colega pede ajuda:

1. Um banner azul aparece no topo do seu Painel com o nome do colega e o contexto.
2. Clique em **Ajudar** para aceitar o pedido.
3. Seu status é atualizado automaticamente para **Ajudando**.
4. Combine com o colega a melhor forma de ajudar (chamada, mensagem, presença física).

Clique em **Dispensar** para fechar o banner sem aceitar (o pedido continua visível para os demais).

## Após a ajuda

Quando o suporte terminar, o desenvolvedor que ajudou deve:

1. Clicar no badge de status no seu card.
2. Selecionar **Ativo** para indicar que voltou à sua tarefa.

## Dicas para um bom pedido

Um contexto claro aumenta a chance de receber ajuda rápida:

**Pouco contexto:** "Travado no ticket VEC-0091"

**Bom contexto:** "Travado no VEC-0091 — a query do Prisma retorna null mesmo com o registro existindo no banco. Já verifiquei o organizationId do middleware e a extensão de tenant."
